# Породично стабло

Статичка веб апликација која приказује јавно, read-only породично стабло и
омогућава посјетиоцима да пошаљу приватне предлоге за исправке.

## Структура

- `family.html` - јавно стабло и панел са објављеним подацима
- `proposal.html` - форма за предлог везана за постојећи `personId`
- `js/family-data.js` - једини извор објављених података стабла
- `js/tree.js` - read-only Family Chart подешавање
- `js/proposal.js` - форма, drag-and-drop и провјера података
- `js/firebase-service.js` - упис предлога у Firestore и слике у Storage
- `firestore.rules` и `storage.rules` - јавни create-only приступ

Предлози се не учитавају на јавним страницама и никада не мијењају
`family-data.js`.

Претрага прихвата име и опционо име оца. Ћирилица, латиница и латиница без
дијакритика се своде на исти облик, па на примјер `Ђуро`, `Đuro` и `Djuro`
дају исти резултат. Резултати приказују очинску линију од најстаријег познатог
претка до пронађене особе.

## Локално покретање

ES modules не треба отварати директно као `file://`. Из коријена пројекта
покрените:

```sh
python3 -m http.server 8000
```

Затим отворите:

```text
http://localhost:8000/family.html
```

Провјера интегритета података:

```sh
node scripts/validate-data.mjs
```

## Firebase подешавање

1. Направите Firebase пројекат и Web app.
2. Укључите Cloud Firestore и Firebase Storage.
3. Копирајте Web app конфигурацију у `js/firebase-config.js`.
4. Инсталирајте Firebase CLI и пријавите се.
5. Повежите локални директоријум са пројектом:

```sh
firebase use --add
```

6. Поставите правила:

```sh
firebase deploy --only firestore:rules,storage
```

7. У Firebase Console укључите App Check за Web app са reCAPTCHA v3.
8. Упишите добијени site key у `appCheckSiteKey` у
   `js/firebase-config.js`.
9. Тек након успјешног теста укључите App Check enforcement за Firestore и
   Storage.

Firebase Web config није приватни серверски кључ. Безбједност зависи од
правила и App Check подешавања.

## Тестирање предлога

1. Отворите `family.html`.
2. Кликните на особу, па у панелу са подацима изаберите
   `Предложи исправку или допуну`.
3. Провјерите да URL садржи постојећи `personId`.
4. Унесите најмање 10 знакова и опционо додајте слику до 5 MB.
5. Пошаљите форму.
6. У Firestore провјерите нови документ у `proposals`.
7. Ако је додата слика, провјерите `proposals/{proposalId}/...` у Storage.

Јавни корисник по правилима може само да направи предлог и отпреми
дозвољену слику. Не може да чита, мијења или брише предлоге.

## Објављивање прихваћених података

Администраторски интерфејс још не постоји. Прихваћене податке ручно унесите у
`js/family-data.js`, уз задржавање постојећих ID вриједности и реципрочних
веза. Након измјене покрените `node scripts/validate-data.mjs`.

Биографија је опционо текстуално поље у `data` објекту особе:

```js
data: {
  gender: "M",
  "first name": "Име",
  "last name": "Презиме",
  birthday: "1900-1980",
  biography: "Први пасус биографије.\n\nДруги пасус биографије.",
  avatar: "",
}
```

Када је `biography` празан или не постоји, секција се не приказује. Текст се
приказује као обичан текст, а празни редови остају видљиви као пасуси.
